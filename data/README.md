# Data — StreamFlix Trial Experiment (Synthetic)

This project uses a **fully synthetic dataset** generated by `src/data/simulate.py`.
There is no external download — the data is reproducible from the simulator.

## Why synthetic?

A custom-designed dataset signals stronger experimentation skill than reusing a
common public dataset. It lets us:

- Define a realistic scenario (StreamFlix trial-to-paid funnel) end-to-end
- Embed a **known ground truth** (true treatment effect, heterogeneity)
- Engineer **realistic edge cases** (SRM bug, page-render glitch, HTE,
  Simpson's paradox setup) so the analysis can detect them
- Include a **pre-experiment covariate** (`prior_watch_hours`) for CUPED
- Demonstrate awareness of metric tiers (primary / secondary / guardrails)

## How to (re)generate the data

From the project root:

```bash
pip install -r requirements.txt
python src/data/simulate.py
```

This writes `data/experiment.csv` (~100,000 rows). The generator is seeded
(`seed=42`) so the dataset is fully reproducible.

## Schema

| Column | Type | Description |
|---|---|---|
| `user_id` | int | Unique user identifier |
| `timestamp` | datetime | When the user started the free trial |
| `group` | str | `control` or `treatment` (assignment) |
| `landing_page` | str | `top_picks` (control variant) or `personalized` (treatment variant) |
| `country` | str | US / UK / CA / AU / Other |
| `device` | str | iOS / Android / Web / TV |
| `source` | str | organic / paid_search / paid_social / referral |
| `is_returning` | bool | True if the user had a prior account |
| `prior_watch_hours` | float | Watch hours in the 30 days before trial (covariate) |
| `converted` | int | 1 if user converted to paid in 14d, else 0 — **PRIMARY** |
| `trial_watch_hours` | float | Total hours watched during trial — secondary |
| `distinct_titles` | int | Distinct titles watched during trial — secondary |
| `day7_active` | int | 1 if active on day 7 of trial — guardrail |
| `page_load_ms` | int | Average homepage load time — guardrail |

## Ground truth (hidden until analysis)

The simulator embeds the following truths that the analysis phases should recover:

- **Overall ATE on conversion:** ~+1–2pp lift (treatment > control)
- **Heterogeneous treatment effect:** ~+2pp for mobile (iOS/Android), ~+0.5pp Web, ~0pp TV
- **Returning users** respond more strongly than new users
- **SRM bug:** Android v<5.1 users intermittently leak from treatment → control
- **Render glitch:** ~0.1% of users see the wrong page vs. their assignment
- **Page load guardrail:** treatment is ~30ms slower (legitimate engineering concern)
- **`prior_watch_hours`** is a strong covariate → enables ~15–30% variance reduction via CUPED
